Password Crack
> 인증 시스템에 대한 무차별 대입 공격을 실습하고, 취약한 패스워드 정책의 위험성을 확인했습니다.
실습 환경 Kali Linux · FTP 서버 · DVWA 웹 서버
다룬 내용
Brute Force Attack — `hydra`의 `-x` 옵션으로 4~6자리 소문자 조합을 탐색해 패스워드를 크랙하고, FTP 서버 로그에 공격 흔적이 남는 것을 확인
Dictionary Attack — 예측 가능한 패스워드 목록(`pw.lst`)을 만들어 `-P` 옵션으로 대입 공격 수행
dpl4hydra — 제조사 기본 계정(삼성·구글·애플 등) 디폴트 패스워드 목록을 생성해 기본 계정의 위험성 확인
DVWA 실습 — 보안 레벨(Low)에서 로그인 동작 분석
배운 점
취약한 패스워드 정책과 변경되지 않은 기본 계정이 얼마나 쉽게 뚫리는지, 그리고 모든 공격 시도가 서버 로그에 기록된다는 점을 직접 확인했다.

Packet Sniffing
> ARP 프로토콜의 구조적 취약점을 이용한 중간자 공격(MITM)을 실습했습니다.
실습 환경 공유기 환경(가정) · 공격자 / 피해자 PC / DVWA 웹서버
다룬 내용
ARP Spoofing 원리 — ARP는 인증 기능 없이 ARP Reply를 신뢰하기 때문에, 공격자가 자신의 MAC을 게이트웨이로 위장하면 동일 네트워크의 트래픽을 가로챌 수 있음
패킷 포워딩 — `fragrouter`로 가로챈 패킷을 실제 게이트웨이로 전달해, 피해자가 공격을 인지하지 못하게 구성
트래픽 분석 — Wireshark의 HTTP stream으로 평문 로그인 정보가 노출되는 것을 확인하고, HTTPS 트래픽은 암호화되어 보이지 않음을 비교
배운 점
인증 없이 ARP Reply를 신뢰하는 구조가 핵심 취약점이며, HTTPS 미사용 사이트는 데이터가 평문 노출된다. 방어책으로 HTTPS 사용과 공용 와이파이 사용 자제가 효과적이라고 정리했다.

Port Scan
> Nmap을 이용한 다양한 포트 스캔 기법을 실습하고, 패킷 수준에서 동작 차이를 분석했습니다.
실습 환경 동일 네트워크의 공격자 / 피해자 PC (Kali Linux)
다룬 스캔 기법
기법	특징
TCP Open Scan	3-way Handshake를 완료, SYN→SYN/ACK→ACK로 포트 개방 확인
TCP Half-Open Scan	SYN/ACK 응답 후 즉시 RST로 연결 종료, 핸드셰이크 미완료
FIN Scan	FIN 패킷 전송, 닫힌 포트만 RST 응답 (열린 포트는 무응답)
XMAS Scan	FIN·PSH·URG 플래그를 동시 설정한 패킷으로 상태 판별
NULL Scan	모든 플래그를 비운 패킷으로 상태 판별
배운 점
Port Scan은 후속 공격을 위한 사전 정찰 도구다. 방어책으로 미사용 포트 차단, 스캔 패턴 탐지·차단, 짧은 간격의 반복 접근 제한을 정리했다.

Snort (IDS)
> 침입 탐지 시스템 Snort의 룰을 직접 작성해, 공격 패킷을 탐지하고 로깅했습니다.
실습 환경 공격자 PC (Kali) · 피해자 PC (Windows) · DVWA
다룬 내용
ICMP 탐지 룰 — `local.rules`에 들어오는 모든 ICMP 패킷을 탐지·경고하는 룰 작성, ping 시 탐지 확인
키워드 탐지 룰 — DVWA 트래픽(80번 포트)에서 `passwd` 키워드를 탐지하는 룰 작성
로그 설정 — `snort.lua` 설정을 수정해 pcap 로그가 정상 저장되도록 구성
XSS 탐지 — Reflected XSS 공격을 수행하고 탐지 로그 확인
배운 점
탐지 룰의 조건(프로토콜·포트·키워드)을 직접 설계하며, IDS가 어떤 기준으로 트래픽을 판단하는지 이해했다. 설정 파일을 직접 디버깅해 로그 출력 문제를 해결한 경험이 남았다.
